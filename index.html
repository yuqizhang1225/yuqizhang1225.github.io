关于无红绿灯路口斑马线礼让问题的思考与构想

最近在上下学的途中，我发现了一个特别值得关注的问题——无红绿灯路口的斑马线礼让情况，真的太让人揪心了。上学路上，我大致观察了一下，有30%到50%的车辆经过斑马线时，既不会停下来礼让行人，甚至都不会减速，依旧疾驰而过。而就在不久前，我看到一则新闻，四川一名和我年纪相仿的高中生，就是在斑马线上等待机动车礼让时不幸发生了事故：双向行驶的车辆中，有一方开了远光灯，强光导致另一方向的司机视线受阻，再加上车速过快，最终酿成了无法挽回的悲剧。看到这则新闻时，我心里又难过又后怕，也忍不住开始思考，能不能找到一些办法，改善这种现状，避免这样的悲剧再发生。

结合平时学到的一些小知识，我突然想到，能不能把闯红灯时用到的压力传感技术，应用到检测车辆是否礼让行人上呢？这个想法虽然还很不成熟，但我还是想试着把它梳理清楚。

我的第一个构想是，在整片斑马线下铺设压力传感器，通过传感器检测到的数值，区分开行人和车辆。如果检测到行人和车辆出现在同一车道，而车辆没有减速，就对车主进行记过处罚。不过我很快就发现了这个方法的漏洞：现在的马路都已经铺设完成，不可能再挖开路面去埋传感器，这样不仅工程量大，还会影响交通。就在我纠结这个问题的时候，我看到了一种超薄触觉阵列的相关介绍，目前已经有多家企业开始量产这种产品，而且它能支持百万级的按压，耐用性应该不成问题。这样一来，我们就不用挖开路面了，直接把超薄触觉阵列铺在道路表面，再刷上一层合适的保护层，既能起到检测作用，又不会破坏路面，也不影响车辆正常行驶。

除了压力传感，我还想到了另一种方法——利用全景相机进行分析。我打算把全景相机安装在斑马线的中央，通过相机构建出三维画面，再让AI对画面进行分析，判断车辆是否礼让行人。其实一开始，我还想过用热成像或者单个监控来检测，但仔细一想，这两种方法很容易出现判断错误。比如热成像和单个监控，很难清晰判断行人和车辆是否在同一车道，很可能会误判，反而达不到效果。而全景相机构建的三维画面，能很好地解决这个层级分析的问题，精准区分行人和车辆的位置，让AI的判断更准确。

不过，我又想到一个问题：就算有了检测方法，要是遇到车主酒驾、疲劳驾驶，或者意识模糊，没有注意到检测提示，依旧不减速通过，甚至即将发生碰撞，该怎么办呢？这时我突然想到了航母上用来为战斗机减速的弹力绳子，能不能模仿这种原理，在检测到即将发生碰撞时，用弹力绳子勾住车轮，强制让车辆减速停下，避免事故发生？

当然，这个构想也存在很多漏洞，我自己也认真思考了两点。第一，万一AI判断错误，强制让车主急刹，会不会导致车主自身发生事故？或者后面的车辆来不及反应，引发追尾？到时候责任该判给谁，也没有明确的标准，也没有人能为车主担保，这是一个很现实的问题。第二，弹力绳子勾住车轮后，如何收回并重复使用？车主下车调整车辆、收回弹力绳时，会不会有安全隐患？不过我也想到，在中国这样比较看重面子的国家，让车主下车亲手收回弹力绳，本身也是一种小小的惩罚，或许能起到一定的警示作用，让车主下次经过斑马线时能更加注意。

![高中女生SolidWorks学习体会 (2)](D:\download\高中女生SolidWorks学习体会 (2).png)

虽然我想到了这些中长期的构想，但从短期来看，这些方法都需要时间去完善、去落地，不能立刻解决问题。所以我觉得，目前最直接有效的办法，要么是增加交警在无红绿灯路口的管理力度，现场引导车辆礼让行人；要么就干脆在这些事故率较高的斑马线处，直接安装红绿灯。毕竟，如果一个需要礼让行人的斑马线，却经常发生事故，无法保障行人的安全，那它存在的意义也就大打折扣了。

作为一名高一学生，我知道自己的这些构想还很稚嫩，可能还有很多考虑不周全的地方，但我真的希望能通过自己的思考，为改善斑马线礼让问题出一份力。我也希望每一位司机都能牢记，斑马线是行人的安全线，礼让行人不仅是法律规定，更是一份责任与善意，愿再也不会有因为不礼让行人而发生的悲剧，让我们上下学的路，能更安全、更安心。

我打算怎么做原型？

我打算怎么做原型？- 原型实现步骤

我把做原型的步骤都想好了，一步步来，肯定能做成！

第一步：画机器狗身体（SolidWorks）

我想设计成这样：

  • 大小：身高大概20cm，身长30cm，跟真小狗差不多，跟着主人走正好
  • 每条腿三个可以动的地方（两个自由度：总共8个，能走路能拐弯）
  • 重心放低一点，这样不容易摔倒
  • 留好摄像头、超声波、电池的位置，方便装
  • 设计成好拆装的，坏了方便改

做好之后导出STL文件，去学校3D打印就好了。

第二步：组装焊接硬件

组装顺序：

  1. 3D打印完，把支撑都清理干净
  2. 把8个舵机装到腿上
  3. 把身体和腿拼起来
  4. 把ESP32、驱动板、传感器都装进去
  5. 装电池，通电，看看有没有冒烟（哈哈，先检查短路）

第三步：写软件驱动，一个个测试

环境先搭好：

  • 装VScode，加ESP32开发板包
  • 装需要的库：PCA9685、ESP32-camera、MPU6050这些

一个个测试，确保每个零件都能用：

  1. 舵机：一个一个转，看看能动，调好中间位置
  2. 摄像头：能拍照，传到电脑看看清楚不清楚
  3. 传感器：超声波能测距离，陀螺仪能读角度，GPS能搜到星
  4. 音频：能录音，能放声音
  5. WiFi：能连上，能打开网站

第四步：写运动控制，让小狗走路

先实现基本动作：

  • 站起来、坐下、前进、左转、右转、停止
  • 调好走路姿势，让它走得稳，不摔倒
  • 速度调到跟人走路差不多，大概每秒走半步，不快不慢

加上避障：

  • 根据AI说的，自动拐弯躲障碍物
  • 超声波测到特别近的东西，直接停下来，不等云端回答，快

第五步：接上AI功能

先申请各种API钥匙：

  • 大模型API
  • 语音API
  • 地图API
  • 把物体识别模型部署好

打通整个流程：

  • 拍照 → 上传 → AI认出来 → 说话告诉你 → 控制小狗拐弯
  • 你说话 → 录音 → 识别 → AI回答 → 说话给你听
  • 你说去哪里 → 导航拿到路线 → 一步步带你走

第六步：整体测试

  1. 在屋里先走一走，试试避障、聊天对不对
  2. 出去到小区里走走，试试导航、认红绿灯
  3. 测测电池能用多久
  4. 开两个小时，看看会不会死机

我觉得可能会碰到这些问题，我想好怎么解决了

我觉得会有什么问题 可能是什么原因 我打算怎么解决
小狗站起来晃，舵机扭力不够 舵机太小，重心太高 换更大扭力舵机，再把重心降一点，身体做轻一点
上传太慢，等半天没反应 照片太大 把照片改小，640×480就够，压得狠一点
AI认不出障碍物，认错了 光线不好，角度不对 我多拍点导盲场景的照片，微调一下模型
走两步就摔了 重心不对，走路姿势不好 调重心位置，慢慢改步态参数，走慢一点
电池很快就没电了 舵机和WiFi一直耗电 不用的时候舵机断电，CPU降频率，省电
外面吵，语音识别不准 噪音大 用降噪麦克风，等你说完再识别，不说的时候不录

我打算这么测试

第一步：我自己先测

每个功能分开试，确保能用，连续开很久看看稳不稳

第二步：找盲人哥哥姐姐测

我已经跟盲校老师说好啦，找3-5个哥哥姐姐试试：

  • 出去走一走，看看避障准不准
  • 用语音方便不方便，好不好学
  • 有没有感觉不安全
  • 问问他们还想要什么功能

我准备了问题问他们：

  • 哪个功能你最喜欢？
  • 哪个功能最难用？
  • 出门你最担心啥？
  • 还想加什么功能？

做成什么样就算原型成功了？

我觉得做到这些就很棒了：

  1. ✅ 能稳稳走路，能躲开障碍物
  2. ✅ 能认出墙、树、人、车这些常见东西
  3. ✅ 能说话聊天，你说指令它听得懂
  4. ✅ 能带你去你想去的地方
  5. ✅ 摔倒了能自动叫救命

原型做好之后呢？

  1. 根据哥哥姐姐说的，改一改不好用的地方
  2. 加上更多功能，比如认人认东西

加油！我觉得我能做成！✨

李飞飞《我看见的世界》读后感

作为一名对人工智能充满好奇，且一直憧憬着未来能出国深造、深耕这一领域的高中生，这个寒假，我认真读完了”人工智能之母”李飞飞教授的传记《我看见的世界》，内心久久不能平静。我不仅对人工智能的底层逻辑有了更清晰、更深刻的认知，更被李飞飞教授的成长经历深深打动，也让我更加坚定了自己追求热爱、奔赴远方的决心，更对未来的深造之路有了更具体的向往。

原生家庭的力量

一方面，我真切感受到原生家庭的力量。父母对孩子的理解、支持与托举，是孩子追逐梦想最坚实的后盾，这一点在李飞飞教授的经历中体现得淋漓尽致。书中有两个细节让我印象尤为深刻：初中时，她热爱阅读课外书，却被老师质疑不务正业，是她的父母力排众议，坚定地站在她这边，坚信广泛阅读能滋养心灵、开阔眼界；为了给她更广阔的发展平台，让她能在更适合的环境中追逐热爱，父母毅然放弃了国内优渥安稳的生活，带着她远赴美国求学，即便到了美国后只能从底层工作做起，省吃俭用艰难支撑家庭，也从未动摇过支持她读书、追求梦想的决心。

热爱与坚守

另一方面，李飞飞教授对人工智能的热爱与持之以恒的努力，更让我看到了追梦路上最动人的模样，也让我更加坚定了深耕这一领域的信念。我一直对人工智能充满浓厚的兴趣，总好奇那些智能算法、图像识别背后的奥秘，而李飞飞教授的经历告诉我，热爱不是一时的新鲜感，而是跨越困境、勇往直前的动力，坚守初心，始终跟随内心的”北极星”笃定前行，才能抵达理想的彼岸。

面对困境的勇气

李飞飞教授15岁初到美国时碰到的种种艰难，远超我想象，更让我心生敬佩。文化隔阂让她难以融入陌生的环境，生活的困顿让她不得不早早体会生活的艰辛，校园霸凌、人身安全威胁等重重困境接踵而至，那些至暗时刻，光是读来就让人揪心。但最让我动容的是，她在书中写道：”这些都是至暗时刻，但它们让我更加珍视在一路上发现的人性的光辉。永远保持好奇心和勇气，永远愿意追问那些看似不可能的问题，追寻自己的北极星。”这句话深深刻在我心里，也让我明白，未来我出国深造，或许也会遇到语言障碍、文化差异、学习压力等困难，但只要保持对人工智能的热爱，带着勇气和好奇心，就没有跨不过去的坎。

结语

李飞飞教授用自己的经历，告诉了我什么是热爱与坚守，也让我对未来的路有了更清晰的方向。作为一名渴望出国深造、深耕人工智能领域的高中生，我深知，这条路不会轻松，但我会以李飞飞教授为榜样，带着勇气和信念，奔赴海外，去探索更广阔的世界，去追寻属于自己的”北极星”。或许这正是人生该有的模样——找到热爱的事物，便为之全力以赴、奋力向前。

一体化智能急救系统设想

今天我想到，急救场景中机器人也能发挥重要作用。

在公共场所最常见的 AED 自动体外除颤仪，虽然配有语音指导，但整个提示流程长达 30–40 秒。再加上现场人员往往处于慌乱状态，很容易错过心脏骤停后的黄金抢救 4 分钟。

因此我设想，可以设计一套一体化智能急救系统：

先将患者安置在预设的定位区域（比如设计成 L 形限位区，让人体被稳定卡在标准位置），再由 AI 自动识别并定位正确的除颤与按压部位，最后通过机械臂自动完成电击、胸外按压、人工呼吸等全套急救操作。

系统优势

这套系统的优势十分明显：

· **安全性提升：**紧急情况下，即便有语音指导，普通人也容易出现操作失误，比如未及时让周围人员远离，可能导致干扰甚至触电风险。而一体化急救设备外部可加装绝缘层，从结构上提升安全性，降低人为失误。

· **避免交叉感染：**在事故现场，如果患者口鼻有血液等体液，人工施救可能存在交叉感染或血型不合带来的健康风险，机器施救则能完全避免这一问题。

· **标准化操作：**我国掌握规范胸外按压技能的人群不足总人口 1%，按压力度、位置不当都会直接导致抢救无效，而标准化的机械急救可以保证操作质量稳定可靠。

延伸方向

在此基础上，我还想到几个延伸方向：

· 软体机器人是否可以应用在这类急救场景中？它更柔软、更贴合人体，或许能减轻胸外按压等操作带来的二次伤害。

· 这套系统能否与其他设备联动，比如附近医院、社区健康监测终端、个人智能健康设备等，实现个人健康数据与急救设备的实时互通，进一步提高抢救成功率？

程序怎么设计？- 软件架构

程序怎么设计？- 我的软件架构

我想了好久，也请教了老师，决定用端云结合的方式，这样ESP32压力小，还能用到最新的AI，太适合我这个项目啦～

整体架构大概这样

为什么要把AI放云端？

  1. ESP32性能不够跑大模型，放云端不用买很贵的硬件
  2. 现在API真的好便宜，调用一次才几分钱，一天用下来也才几块钱
  3. 以后AI更好了，不用换硬件，直接用新模型就行，方便升级

我分了四层，好打理

1. 最底层：驱动各个硬件（都在ESP32上）

  • 摄像头驱动：拍好照片，压缩成小JPEG传给云端
  • 传感器驱动：读超声波、陀螺仪、GPS的数据
  • 舵机驱动：控制十二个舵机，让小狗走路拐弯
  • 音频驱动：录下你说的话，放出小狗说的话

2. 本地控制：还是ESP32

  • 控制小狗：前进、拐弯、停止、坐下
  • 紧急停车：超声波检测到特别近的东西，直接停车，不用等云端回消息，更快更安全
  • 摔倒检测：陀螺仪看出来摔了，直接叫救命

3. 云端AI：各种AI能力

模块 做什么 我打算用什么
语音识别 把你说的话变成文字 百度/阿里语音识别，免费额度够我用了
语音合成 把文字变成声音读出来 和语音识别一起，接口就有
物体识别 认出前面是什么障碍物、红绿灯 YOLOv5n 小模型，挺快的
AI聊天 陪你说话，理解你的指令 通义千问API，便宜好用
导航 找路线，带你去目的地 高德地图API，现成的

4. 手机APP（可选，以后做）

  • 连WiFi，设置紧急联系人
  • 收到求助提醒，看你在哪里
  • 远程更更新程序

刚接触SolidWorks时，我完全是个“门外汉”。面对复杂的软件界面，密密麻麻的工具栏、陌生的专业术语，还有那让人摸不着头绪的“设计树”，我常常对着屏幕发呆，连新建一个零件文件都要摸索半天。老师说，草图是建模的根基，可我最初画的草图，总是蓝莹莹的一片，老师告诉我，这是“欠定义”的表现，就像没有根基的房子，随时可能“倒塌”。我试着按照老师的方法，添加几何关系、标注尺寸，一遍又一遍调整，直到草图变成黑色的“完全定义”状态，那一刻，小小的成就感在心底悄悄发芽。

学习的过程中，困难远比我想象的多。刚开始练习拉伸、切除这些基础特征时，我总爱凭着感觉来，草图不规范、特征顺序混乱，常常出现模型变形、报错的情况。有一次，我想画一个简单的水杯模型，先做了倒角再拉伸基体，结果修改尺寸时，整个模型直接“爆红”，所有的努力都白费了。我又急又气，甚至想过放弃，可想起自己要做出专属模型的心愿，又咬牙重新开始。我对照着教程，理清建模逻辑，遵循“先主体后细节”的顺序，先拉伸出水杯的主体，再添加圆角、抽壳等细节特征，这一次，模型终于顺利成型。原来，SolidWorks和我们学习数学一样，讲究逻辑和严谨，一步错，步步错，唯有静下心来，循序渐进，才能做好每一个步骤。

随着学习的深入，我慢慢走出了新手的困境，也逐渐发现了建模的乐趣。我开始尝试从身边的事物入手，画课本、画文具、画家里的小摆件。从简单的长方体、圆柱体，到复杂一点的台灯、小盒子，每一个模型的完成，都是一次成长的见证。我还记得第一次尝试装配体建模时，不知道如何给零件添加配合，只能凭着感觉拖动零件，结果装配体一动就乱，后来我认真学习了重合、同心、平行等配合方式，慢慢摸索出技巧，当我成功将几个零件组装成一个完整的小玩具时，那种成就感难以言喻。

这段学习经历，也让我改掉了粗心大意的毛病。以前做事情总是急于求成，忽略细节，可在建模时，哪怕是一个小小的尺寸误差、一个遗漏的几何关系，都可能导致整个模型失败。有一次，我因为一时疏忽，把零件的尺寸标错了，导致后续的装配无法进行，只能重新修改所有相关特征。这件事让我明白，严谨不仅是建模的要求，更是做人做事的准则。现在的我，做任何事情都会多一份细心和耐心，遇到问题不再急于求成，而是静下心来分析原因，一点点解决。

（加装配、等的组图）

有人说，SolidWorks是工程师的工具，不适合我们高中生学习，可我却不这么认为。作为一名高一学生，我们正处于好奇心旺盛、求知欲强烈的阶段，学习SolidWorks，不仅让我掌握了一项新技能，更锻炼了我的空间想象力和逻辑思维能力，这对我的数学、物理学习也有很大的帮助。我还学会了利用快捷键、模板等工具提高建模效率，也懂得了遇到问题时，善用软件自带的帮助功能和搜索引擎，不再盲目求助。

如今，我已经能独立完成一些简单的零件建模和装配体设计，虽然距离精通还有很长的路要走，但我已经深深爱上了这种“从无到有”的创造过程。学习SolidWorks的这段日子，有困惑、有挫折、有汗水，但更多的是收获与成长。它让我明白，任何技能的掌握都没有捷径，唯有脚踏实地、持之以恒，才能不断进步。

未来，我希望能继续深入学习SolidWorks，尝试设计更复杂的模型，把更多天马行空的想法变成现实。我也想告诉和我一样对建模感兴趣的同学，不要害怕困难，勇敢迈出第一步，只要有耐心、有坚持，就一定能在建模的世界里收获属于自己的精彩。这段与SolidWorks相伴的时光，终将成为我高中生活中一段难忘而珍贵的回忆。
